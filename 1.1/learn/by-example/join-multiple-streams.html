Join Multiple Streams

Streams can be joined using a streaming query logic. In this example, events related to raw materials and production output wait for 10 seconds and then correlate the data gathered during that time period. An alert is triggered if the raw material usage is 5% higher than the production outcome. This example can be extended to make decisions based on the output of the streaming rules. Ballerina Streams is an experimental feature. Use the --experimental flag when compiling Ballerina files, which have streaming constructs.

import ballerina/http;

type ProductMaterial record {
    string name;
    float amount;
};

type MaterialUsage record {
    string name;
    float totalRawMaterial;
    float totalConsumption;
};

stream<ProductMaterial> rawMaterialStream = new;
stream<ProductMaterial> productionInputStream = new;

These are the input streams that use ProductMaterial as the constraint type.

 
stream<MaterialUsage> materialUsageStream = new;

This is the output stream, which contains the events/alerts that are generated based on the streaming logic.

 
function initRealtimeProductionAlert() returns () {

    materialUsageStream.subscribe(printMaterialUsageAlert);

Whenever the materialUsageStream stream receives an event from the streaming rules defined in the forever block, the printMaterialUsageAlert function is invoked.

Gathers events related to raw materials through the rawMaterialStream stream and production-related events through the productionInputStream. The raw materials usage and production outcome for the last 10 seconds are calculated and an alert is triggered if the raw material usage is 5% higher than the production outcome. This forever block is executed once when initializing the service. The processing happens asynchronously each time the requestStream or productionInputStream receives an event.

 
    forever {
        from productionInputStream window time(10000) as p
        join rawMaterialStream window time(10000) as r
        on r.name == p.name
        select r.name, sum(r.amount) as totalRawMaterial,
                sum(p.amount) as totalConsumption
        group by r.name
        having ((totalRawMaterial - totalConsumption) * 100.0 /
                totalRawMaterial) > 5
        => (MaterialUsage[] materialUsages) {

            foreach var usage in materialUsages {
                materialUsageStream.publish(usage);
            }
        }
    }
}

materialUsages is the output that matches the defined streaming rules. It is published to the materialUsageStream stream. The selected clause should match the structure of the MaterialUsage type.

 
function printMaterialUsageAlert(MaterialUsage materialUsage) {
    float materialUsageDifference = (materialUsage.totalRawMaterial -
            materialUsage.totalConsumption) * 100.0 /
                (materialUsage.totalRawMaterial);

    io:println("ALERT!! : Material usage is higher than the expected",
            " limit for material : ", materialUsage.name,
            ", usage difference (%) : ", materialUsageDifference);
}

# To run this sample, navigate to the directory that contains the
# `.bal` file, and execute the `ballerina run` command.
$ ballerina run --experimental join_multiple_streams.bal
[ballerina/http] started HTTP/WS listener 0.0.0.0:9090

# Execute the following cURL request. 
$ curl -H "Content-Type: application/json" -X POST -d "{'name':'Teak','amount':1000.0}" http://localhost:9090/rawmaterial
{"message":"Raw material request successfully received"}
# Run the following cURL request within 10 seconds of running the first cURL request.
$ curl -H "Content-Type: application/json" -X POST -d "{'name':'Teak','amount':500.0}" http://localhost:9090/productionmaterial
{"message":"Production input request successfully received"}

# If the two cURL requests were executed within 10 seconds, the following log is printed where you ran the 'join_multiple_streams.bal'.
