Using GraphQL and Ballerina with multiple data sources

In today’s digital transformation era, making applications and web services talk to each other is unavoidable and APIs allow communication between those applications. Various protocols and specifications will result in the API architecture at the end, defining the semantics and syntax of how the messages are passed across the wire.

In this InfoQ article, WSO2’s Director of Engineering, Anupama Pathirage, explores how we can expose the data in a MySQL database as an API using GraphQL with Ballerina.

Efficient Joins with Ballerina Query Expressions

Query Expressions in Ballerina allow you to write queries for data processing using SQL-like syntax. With this language-integrated query feature, Ballerina offers you more quality in requesting your data faster and more accurately. In this article, WSO2 Senior Software Engineer, Lasini Liyanage, provides a walkthrough on how to use Joins with Ballerina Query Expressions.

Create a data API with Ballerina and H2

This quick tutorial by WSO2’s Associate Director - Developer Advocate Dakshitha Ratnayake, shows you how to easily create a REST API to perform some basic database queries on an H2 database using Ballerina.

For more great content on Ballerina, browse through a selection of articles on the WSO2 Stack Overflow Collective.

Latest release: Ballerina 2201.1.1 (Swan Lake Update 1)

Swan Lake 2201.1.1 is the first patch release of Ballerina 2201.1.0 (Swan Lake Update 1) and includes a new set of bug fixes to the compiler, runtime, standard library, and developer tooling.

To see all the changes and improvements we’ve made so far, read our release note.
