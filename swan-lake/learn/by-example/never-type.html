Never Type

The never type represents the type which has no values. This can be useful to describe the return type of a function, if the function never returns. As no value can ever belong to the never type, variables of type never cannot be declared.

This is an example of using never as the type parameter in xml. xml<never> describes the xml type that has no constituents, i.e., the empty xml value.

 
    map<never> someMap = {};

You can define a mapping value with never as its constraint. But you can never add members to this map.

 
    table<Person> key<never> personTable = table [
        {name: "John", age: 23},
        {name: "Paul", age: 25}
    ];
}

You can specify a key-less table with the never type as the key constraint.
