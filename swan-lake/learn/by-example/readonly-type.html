Readonly Type

The readonly type consists of values whose read-only bit is set. (), boolean, int, float, decimal, string, error, function, service, and typedesc are considered to be inherently immutable basic types, and a value belonging to an inherently immutable basic type will always have its read-only bit set, and thus will always belong to the readonly type. XML, lists, mappings, tables, and objects are selectively immutable types, where values belonging to such types are immutable and belong to the readonly type only if their read-only bit is set. A readonly value is deeply immutable, and guarantees that members, if any, are also immutable.

A value belonging to an inherently immutable basic type can be used directly where an immutable readonly value is expected.

 
    string ballerina = "ballerina";
    readonly b = ballerina;
    io:println(b);

    map<int> & readonly marks = {
        math: 80,
        physics: 85,
        chemistry: 75
    };

A value of a selectively immutable type can be created as immutable, by using an intersection type with readonly as the contextually expected type.

 
    readonly c = marks;

If the inherent type of a value is T & readonly, its read-only bit is set, and the value will belong to the readonly type.

 
    io:println(c);
    io:println(marks.isReadOnly());
}

The .isReadOnly() check for an immutable value returns true.
