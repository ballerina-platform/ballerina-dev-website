Function Mocking

Mock functions allow you to hide the real functions and engage your own functions when running tests. This allows you to isolate your test functions from the rest. Both mocking an imported function and a function declared in the module under test can be mocked with dedicated annotations.

Mocking an imported function will apply the mocked function to every instance of the original function call. It is not limited to the test file where it is being mocked. Mocking a function from the module under test can be scoped to have different behaviors for different test cases.

For more information on the underlying module, see The Test Module.

This is the global array to which the mock function will push elements.

    moduleName: "ballerina/io",
    functionName: "println"

function mockIoPrintLn((any|error)... text) {

This specifies a mock function that should replace the io:println function. The mock function signature should match that of the real.

 
    outputs.push(text);
}

This line adds the value to an array instead printing it.

 
public function printMathConsts() {
   io:println("Value of PI : ", math:PI);

The function makes a call to io:println which should be replaced with the mock function.

 
@test:Config {}
function testMathConsts() {
   printMathConsts();
   test:assertEquals(outputs[0].toString(), "Value of PI :  3.141592653589793");
}

This specifies the test function that tests printMathConsts function.

This is the main_test.bal file of the example for mocking a function from the module under test.

 
@test:MockFn { functionName: "intAdd" }

This creates an object for stubbing calls to intAdd.

 
test:MockFunction intAddMockFn = new();

This object will be used to stub the calls to intAdd function.

 
@test:Config {}
function testAssertIntEquals1() {

    test:when(intAddMockFn).thenReturn(20);
    test:assertEquals(intAdd(10, 6), 20, msg = "function mocking failed");
}

This stubs the calls to intAdd function to return the specified value.

 
@test:Config {}
function testAssertIntEquals2() {

    test:when(intAddMockFn).thenReturn(20);

This stubs the calls to intAdd function to return the specified value.

 
    test:when(intAddMockFn).withArguments(0, 0).thenReturn(-1);

This stubs the calls to intAdd function to return the specified value if the specified arguments are provided.

 
    test:assertEquals(intAdd(10, 6), 20, msg = "function mocking failed");

This should return 20.

 
    test:assertEquals(intAdd(0, 0), -1,
        msg = "function mocking with arguments failed");

This should return -1.

 
}

public function mockIntAdd(int a, int b) returns int {
    io:println("I'm the mock function!");
    return (a - b);
}

This is the mock function that should be called in place of intAdd function. The mock function signature should match the intAdd function.

 
@test:Config {}
function testAssertIntEquals3() {

    test:when(intAddMockFn).call("mockIntAdd");

This stubs the calls to intAdd function to invoke the created mock function.

 
    test:assertEquals(addValues(10, 6), 4, msg = "function mocking failed");

This verifies that the mock function is called in place of the real.

 
}
