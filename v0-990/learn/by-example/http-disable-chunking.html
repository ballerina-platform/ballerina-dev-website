Disable Chunking

This sample demonstrates how to change the Ballerina HTTP client connector’s chunking behavior. By default, the HTTP client sends content-length messages. If the message size is larger than the buffer size (8K), messages are chunked. Chunking can be disabled using the connector options.

The HTTP client’s chunking behaviour can be configured as auto, always, or never. In this example, it is set to as never, which means that chunking never happens irrespective of how it is specified in the request. When chunking is set to auto, chunking is done as specified in the request.
